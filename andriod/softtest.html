知识归纳

缺陷的定义

  1. 软件没有实现产品规格说明书所要求的功能模块
  2. 软件中出现了产品规格说明指明不应该出现的错误
  3. 软件实现了产品规格说明没有提到的功能模块
  4. 软件没有实现瑞士产品规格说明没有明确提及但应该实现的目标
  5. 软件难以理解，不容易使用，运行缓慢

缺陷的等级

  1. 致命级(系统崩溃、死机)
  2. 严重级(部分功能丧失)
  3. 一般级(功能没有完全实现，但不影响使用)
  4. 建议级(处理测试人员提出对茶农特性改进的建议或质疑)

软件测试的目的

测试的目的是确保软件系统的质量、正确性和可靠性，，满足用户需求，并提供良好的用户体验。通过测试，可以及时发现和修复缺陷，减少风险，提高软件的可信度和可用性。

软件测试的分类

A、按照开发阶段划分

  1. 单元测试

    单元测试又称模块测试，对软件最小可测单元进行检查。单元测试需要从软件的内部结构出发设计测试用例。多个模块可以并行独立地进行测试。

  2. 集成测试

    将所有的程序模块进行有序、递增的测试，检验程序单元或部件的接口关系

  3. 系统测试

    系统测试是对集成的硬件和软件系统进行的测试，在真实或模拟系统运行的环境下，检查完整的程序系统能否和系统正确配置、连接，并满足用户需求。

  4. 验收测试

    按照项目任务或合同，供需双方签订验收依据文档进行的对整个系统的测试与评审觉得是否接受或拒收系统。

B、按照测试技术划分

  1. 白盒测试

    白盒测试又叫结构性测试或逻辑驱动测试

  2. 黑盒测试

    黑盒测试又叫功能测试或数据驱动测试

  3. 灰盒测试

    介于黑白黑白盒测试之间的测试，既关注输出对输入的正确性，又关注内部表现。

C、按照被测软件是否实际运行划分

  1. 静态测试

    静态测试包括代码检查、结构分析、代码质量度量

  2. 动态测试

    通过输入测试用例对其运行情况进行检测，即分析输入输出的对应关系，达到测试的目的

D、按照测试实施组件划分

  1. 开发方测试

    通常也叫“验收测试”和“α测试”

  2. 用户测试

    β测试

  3. 第三方测试

    介于开发方与用户方之间的测试组织，也称独立测试

软件质量模型中有哪些质量特性

  1. 功能性
  2. 可靠性
  3. 易用性
  4. 效率
  5. 维护性
  6. 可移植性

静态测试与动态测试概念

静态测试：不实际运行被测软件，而只是静态检查程序代码、界面或文档中可能存在的错误的过程

动态测试：实际运行被测试程序，输入相应的测试数据，检查实际输出结果和预期结果是否符合的过程

两者关系：相辅相成，可以在软件开发过程中结合使用，提高软件的质量和可靠性

测试用例的核心

概念核心：对被测对象的特定功能或行为进行验证或评估的一组输入、预期输出和执行步骤的描述。测试用例是软件测试过程中的基本单元，用于检查软件是否符合预期的要求和规范

核心要素：输入数据、预期输出、执行步骤、预期结果

黑盒测试的方法

  1. 等价类方法
  2. 边界值分析
  3. 决策表
  4. 因果图
  5. 场景法
  6. 状态迁移法

决策表怎么来做

  1. 列出所有的条件桩和动作桩
  2. 确定规则的个数
  3. 填入条件项
  4. 填入动作项，得到初始决策表
  5. 简化决策表，合并相似规则

逻辑覆盖有那几种

  1. 语句覆盖
  2. 判定覆盖
  3. 条件覆盖
  4. 路径覆盖
  5. 判定-条件覆盖
  6. 组合-条件覆盖

路径覆盖方法步骤

  1. 理解软件系统
  2. 识别可行路径
  3. 划分路径集合
  4. 设计测试用例，执行测试用例
  5. 分享测试结果
  6. 完善测试覆盖

单元测试概念

单元测试是软件开发过程种的一种测试方法，用于验证软件中最小可测单元的正确性。它的目标是隔离和测试单个功能模块，以确保其按照预期行为工作

特点：独立性、自动化、可重复性、细粒度、高覆盖率

单元测试的内容

  1. 模块接口测试
  2. 模块局部数据结构测试
  3. 模块编辑条件测试
  4. 模块中所有独立执行通路测试
  5. 模块的各条错误处理通路测试

单元测试的方法

  1. 规格导出法
  2. 等价类划分法
  3. 边界分析法
  4. 状态迁移测试法
  5. 分支测试法
  6. 条件测试法
  7. 内部边界值测试错误猜测法
  8. 循环测试法

集成测试

目的：确保各个单元组合在一起后能偶按既定意图协作运行，并确保增量的行为正确

测试方法：静态和动态测试技术，使用灰盒测试用例设计方法。

意义：集成测试阶段测试方法时动态变化的，从白盒测试方法向黑盒测试方法逐渐过渡
